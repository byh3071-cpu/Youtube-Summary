# Focus Feed · AI 기능 시야 확장

요약·인사이트를 넘어, **입력·출력·대상·시점·형태**를 넓혀 본 AI 적용 방향입니다.  
해커톤 아이디어, 장기 로드맵, 피처 브레인스토밍용 참고 문서입니다.

---

## 1. 입력(Input) 다양화

| 방향 | 설명 | 예시 |
|------|------|------|
| **음성** | "이 영상 1분 요약해줘" 음성 명령 | 보행/운전 중 핸즈프리로 요약 요청 |
| **화면/스크린샷** | 지금 보는 화면 캡처 → "이 부분 요약/설명해줘" | 강의 슬라이드, 코드 화면 해석 |
| **타임스탬프** | "5:30~8:00 이 구간만 요약" | 긴 영상 중 관심 구간만 집중 |
| **링크/붙여넣기** | URL 하나 넣으면 채널·시리즈·관련 영상까지 맥락 분석 | "이 영상이 어떤 시리즈에 속하는지" |
| **질문** | "SaaS 마케팅 조언 나와?" → 요약/자막 기반 답변 | Q&A형 인터페이스 |
| **목표 문장** | 이미 있음 (My Focus) | "3개월 안에 1인 SaaS" → 오늘 볼 것 추천 |

→ **독창 포인트**: “텍스트 요약 요청”만이 아니라 **언제, 어떤 형태로** AI에게 물어보는지가 제품 차별화가 됨.

---

## 2. 출력(Output) 다양화

| 방향 | 설명 | 예시 |
|------|------|------|
| **행동 제안** | 이미 있음 (rank 시 action) | "이 영상 보고 나서 할 일 한 가지" |
| **비교** | "A 영상 vs B 영상 입장 차이" | 두 채널/두 영상 대조 요약 |
| **타임라인** | "주장이 나오는 시점" 리스트 | 3:00 주장1, 7:20 주장2 … |
| **논점 정리** | 찬반/논쟁 영상 → 대립 구조로 정리 | "찬성: … / 반대: …" |
| **한 문장/한 단어** | 초단축 표현 | "한 줄: ~", "한 단어: ~" |
| **다른 포맷으로 변환** | 요약 → 트윗 초안, 블로그 개요, 메모, 플래시카드 | 콘텐츠 재가공 |
| **챕터/구간 제목** | 자막·타임스탬프 기반 AI 챕터 | "이 구간: ~" (유튜브 자체 챕터 없을 때) |
| **시각** | 요약/키워드 → OG 이미지, 인포그래픽, 단일 프레임 추천 | 공유용 카드, 썸네일 대안 |

→ **독창 포인트**: “요약 텍스트”만 내보내지 않고 **다음 행동·다른 포맷·시각**까지 AI가 제안하면 “도구”가 아니라 “조수”처럼 느껴짐.

---

## 3. 대상/역할(User & Use Case) 확장

| 대상 | AI가 해주는 것 |
|------|----------------|
| **학습자** | 난이도 표시, 선수 지식 추천, "이 영상 전에 볼 것" |
| **출퇴근** | "오늘 20분이면 들을 수 있는 것", 라디오 모드용 구간 하이라이트 |
| **팀/조직** | "이번 주 팀이 봐야 할 영상 3개" + 왜 봐야 하는지 한 줄 |
| **크리에이터** | "내 영상 요약/챕터 제안", "숏스용 클립 구간 제안" |
| **비원어민** | 요약 + 핵심 용어 설명, 난이도(어휘/속도) |
| **집중이 어려운 사용자** | "주의가 필요한 구간", "건너뛰어도 되는 구간" |

→ **독창 포인트**: “일반 시청자”만이 아니라 **학습자·팀·크리에이터·접근성**까지 넣으면 타깃이 분명해지고 스토리가 살아남.

---

## 4. 시점(Temporal) 확장

| 시점 | AI 기능 |
|------|----------|
| **보기 전** | 한 줄 예고, "이 영상에서 얻을 수 있는 것", 난이도·소요 시간 |
| **보는 중** | 구간별 요약, "지금 나오는 내용 한 줄", 실시간 Q&A(캐시 기반) |
| **본 직후** | 요약·인사이트(이미 있음), "다음에 볼 영상", 메모/플래시카드 생성 |
| **나중** | "저번에 본 그 영상에서 OOO 나왔지?" 검색, 주/월 브리핑 |

→ **독창 포인트**: “본 뒤 요약”만이 아니라 **전·중·후·나중** 전 구간에 AI가 끼어들 수 있음.

---

## 5. 멀티모달 & 시각

| 방향 | 설명 |
|------|------|
| **키 프레임** | 구간 대표 이미지 추출 → "이 구간은 이 장면" |
| **요약 → 이미지** | 요약 문장으로 OG/썸네일 생성 (Imagen 등) |
| **차트/데이터** | 영상에서 언급된 숫자·비교 → 간단 차트/표 정리 |
| **자막 하이라이트** | 중요 문장만 색/굵기로 강조 (요약 기반) |

→ **독창 포인트**: “텍스트만”이 아니라 **이미지·차트·하이라이트**까지 나가면 시연 시 임팩트가 큼.

---

## 6. 개인화 심화

| 방향 | 설명 |
|------|------|
| **선호 형식** | 불릿 vs 문단, 짧은 vs 긴 요약 (설정 또는 학습) |
| **관심 키워드** | 특정 키워드 포함 구간만 하이라이트/요약 |
| **집중 시간** | "5분 버전 요약", "20분 버전 요약" (길이에 맞춰 구간 추천) |
| **학습 스타일** | 예시 위주 vs 개념 위주 요약 톤 조정 |

→ **독창 포인트**: “목표 기반 추천”을 넘어 **읽기/보기 스타일**까지 반영하면 개인화 스토리가 강해짐.

---

## 7. 콘텐츠 간 연결(Cross-Content)

| 방향 | 설명 |
|------|------|
| **시리즈/순서** | "이 영상 전에 보면 좋은 것", "다음 단계 영상" |
| **채널 간 비교** | "A채널은 ~, B채널은 ~" 주장 요약 |
| **테마 묶음** | 여러 영상에서 "SaaS 마케팅"만 뽑아 한 번에 요약 |
| **공백 채우기** | "이 주제에서 아직 안 본 관점" 추천 |

→ **독창 포인트**: “한 영상” 단위를 넘어 **여러 영상·채널·테마**를 엮는 AI가 차별점이 됨.

---

## 8. 생산성 & 재가공

| 방향 | 설명 |
|------|------|
| **트윗/포스트 초안** | 요약 → SNS용 1~2문장 초안 |
| **회의 준비** | "이 영상 보고 회의에서 논의할 질문 3개" |
| **메모/노트** | 요약 + 타임스탬프 → Notion/노트 형식 내보내기 |
| **플래시카드** | 핵심 개념 → Q&A 카드 생성 |

→ **독창 포인트**: “보고 끝”이 아니라 **다른 도구·상황으로 이어지는** AI 파이프라인.

---

## 9. 발견(Discovery) & 신뢰

| 방향 | 설명 |
|------|------|
| **논란/주장 정리** | "이 영상에서 주장하는 바", "반론이 있을 수 있는 부분" |
| **출처/근거** | "이 수치는 어디서 나왔는지" (자막 기준 힌트) |
| **트리거/난이도** | 감정 부담, 전문 용어 많음 등 태그 |
| **갭 추천** | "지금까지 본 것과 연결되는 다음 영상" |

→ **독창 포인트**: “많이 추천”이 아니라 **무엇을 믿고, 무엇을 조심해 볼지**까지 AI가 알려주는 방향.

---

## 10. 접근성 & 언어

| 방향 | 설명 |
|------|------|
| **요약 번역** | 한국어 요약 → 영어 한 줄 (또는 반대) |
| **용어 풀어쓰기** | 전문 용어 옆에 한 줄 설명 |
| **속도/난이도** | "빠른 말 많음", "초보자도 이해 가능" |
| **키드 세이프** | 부적절 구간 제외 요약 (정책·모델 한계 명시) |

→ **독창 포인트**: “요약 존재”만으로도 접근성이 올라가고, **언어·난이도·용어**까지 보조하면 포용적 제품 스토리가 됨.

---

## 정리: 시야를 넓힌 AI 포지셔닝

- **입력**: 텍스트뿐 아니라 음성, 화면, 구간, 질문, 목표.
- **출력**: 요약뿐 아니라 행동, 비교, 타임라인, 챕터, 다른 포맷, 시각.
- **대상**: 시청자뿐 아니라 학습자, 팀, 크리에이터, 비원어민.
- **시점**: 본 뒤뿐 아니라 보기 전, 보는 중, 나중.
- **형태**: 텍스트뿐 아니라 이미지, 차트, 하이라이트.

이 중 **한두 축만** 골라서 “우리는 이 축에서 AI를 이렇게 쓴다”고 명확히 하면, 해커톤에서도 제품 비전에서도 **다양하고 독창적인 AI**로 보이기 좋습니다.
