# Cursor가 하는 동안 Antigravity/사용자가 할 수 있는 작업

Cursor가 코드를 작성·수정하는 동안, **Antigravity(제미나이)** 또는 **사용자**가 병렬로 진행할 수 있는 작업 목록입니다. 코드 변경 없이 기획·문서·QA 준비만으로 진행할 수 있습니다.

---

## Antigravity가 할 수 있는 작업

### 1. Phase 4 QA 체크리스트 미리 정리
- [ ] **라디오 모드 전용 체크리스트** 작성  
  - 예: "라디오에 추가" 클릭 → 하단 플레이어 노출, 재생/일시정지, 다음/이전, 닫기 각각 한 번씩 확인  
  - 모바일: 사용자 터치 후 재생 시작 여부  
  - 탭 전환 후에도 오디오 유지 여부  
- [ ] 완료 후 `AI_COLLABORATION.md` 브라우저 검증 체크리스트에 "라디오 플레이어" 항목 반영

### 2. 다음 Phase(5) 기획 초안
- [ ] Phase 4 완료 후 다음에 넣을 기능 후보 정리  
  - 예: TTS로 요약 읽어주기, 오늘의 인사이트 3줄, 큐 저장/불러오기 등  
- [ ] 우선순위(즉시 / 이번 사이클 / 후속 개선) 초안만 적어 두기

### 3. Handoff 받을 준비
- [ ] Cursor 완료 시 전달받을 **"확인 요청"** 항목 미리 읽어 두기  
  - `CURSOR_HANDOFF.md`의 "Antigravity에게 전달할 요약" 블록  
- [ ] 에러 발생 시 사용자에게 요청할 **로그 수집 방법** 문구 준비  
  - 예: "브라우저에서 F12 → Console 탭에서 빨간색 에러 메시지 캡처나 복사해 주세요."

### 4. 사용자 스토리/완료 기준 문서화
- [ ] "라디오처럼 틀어놓고 일한다" 시나리오를 1~2문장으로 정리  
- [ ] `AI_COLLABORATION.md` Handoff 템플릿에 이번 사이클 **완료 기준** 채우기

---

## 사용자가 할 수 있는 작업

### 1. 환경/설정 점검
- [ ] `.env.local`에 `YOUTUBE_API_KEY`, `GEMINI_API_KEY` 등 필요한 키가 들어 있는지 확인  
- [ ] `npm run dev` 한 번 실행해 두어서 Cursor 작업 후 바로 브라우저 확인 가능하게 하기

### 2. QA 시나리오 생각해 두기
- [ ] 자주 쓰는 유튜브 채널/영상 1~2개 정해 두기  
- [ ] "라디오에 추가 → 재생 → 다음 곡 → 탭 전환" 같은 흐름을 한 번만 적어 두기  

### 3. 에러 났을 때 공유할 것 준비
- [ ] 브라우저 개발자 도구(F12) → Console 탭 여는 법만 알고 있기  
- [ ] 에러 메시지가 뜨면 화면 캡처 또는 텍스트 복사해서 Antigravity에게 전달할 수 있게 하기

### 4. 다음에 넣고 싶은 기능 메모
- [ ] "나중에 이거 넣고 싶다" 하는 기능을 짧게라도 적어 두기  
  - Antigravity가 Phase 5 기획할 때 참고 가능

---

## 참고

- **Cursor**: 코드 구현, 빌드/린트, 기술적 검증  
- **Antigravity**: 기획, 우선순위, 완료 기준, 브라우저 QA, 사용자 안내  
- Cursor 작업이 끝나면 `CURSOR_HANDOFF.md`의 **Cursor → Antigravity 핸드오프** 절차대로 이어받으면 됩니다.
