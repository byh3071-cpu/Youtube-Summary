# 수익화 타깃 vs 현재 프로젝트 정렬 · 확장 포인트

수익화 시나리오(개인 / 팀·기업 / B2B·크리에이터)에 맞게 **지금 프로젝트가 어떻게 설정돼 있는지**, 그리고 **어디를 확장하면 되는지** 정리한 문서입니다.

---

## 1. 개인 타깃 (1인 창업자, 직장인, 대학원생)

### 1-1. 현재 맞게 설정된 부분 ✅

| 기능 | 현재 상태 | 타깃과의 정렬 |
|------|------------|----------------|
| **구글 로그인** | Supabase Auth, Google OAuth | 개인 계정 단위 사용 가능 |
| **북마크** | 로그인 시에만 저장, `user_id`로 개인별 관리 | 개인별 저장·동기화에 적합 |
| **커스텀 소스(채널 추가)** | 로그인 사용자만 저장, `user_id`로 개인별 | “내가 보는 채널만”에 적합 |
| **피드·카테고리·키워드 필터** | 유튜브+RSS 통합, AI·자기계발 등 필터 | 학습·업무용 큐레이션에 적합 |
| **My Focus(목표 기반 브리핑)** | 목표 입력 → AI가 오늘 볼 영상 추천 | “시간 아끼기·목표 맞춤”에 적합 |
| **AI 3줄 요약·인사이트** | 자막/제목·설명 기반, 품질 평가·재생성 | “빠르게 핵심만”에 적합 |
| **라디오(백그라운드 재생)** | 큐에 넣고 오디오만 재생 | 출퇴근·집중 작업 시나리오에 적합 |
| **롱폼/숏폼 리얼 뷰** | 길이별 필터·세로 스크롤 | “짧은 것만” / “긴 강의만” 선택에 적합 |

→ **정리**: 개인 타깃의 **핵심 시나리오(피드 큐레이션, AI 요약·브리핑, 북마크, 라디오)**는 이미 구현돼 있고, 로그인·개인별 데이터(북마크·커스텀 소스)도 갖춰져 있어 **개인 수익화(Free vs Pro)의 기반**으로 쓰기 좋습니다.

### 1-2. 개인 타깃 쪽에서 부족한 부분 · 확장 포인트

| 부족한 점 | 확장 방향 (우선순위) |
|-----------|----------------------|
| **요약/인사이트/브리핑에 사용자별 제한 없음** | Free: 일 N회 요약·인사이트 제한, 브리핑 주 1회 등. Pro: 무제한. **로그인 사용자 기준으로 일일/주간 사용량을 DB나 Redis에 적재 후 제한**하면 됨. |
| **플레이리스트가 사용자별이 아님** | `playlists` 테이블에 `user_id` 없음 → **`user_id` 컬럼 추가**, 저장/조회 시 로그인 사용자만 본인 플레이리스트만 보게 하면 개인 타깃과 정렬됨. (미로그인은 저장 불가 또는 로컬만) |
| **Pro 전용 기능이 없음** | 챕터/구간 요약, “오늘 한 줄 브리핑” 등은 **기능을 먼저 구현한 뒤**, 플랜(Free/Pro) 체크만 추가하면 됨. DB에 `subscriptions` 또는 `user_plan` 테이블 도입 후, API/액션에서 플랜별 제한 분기. |
| **목표(My Focus)가 로컬만** | `goals`가 localStorage → **로그인 시 Supabase 등에 저장·동기화**하면 기기 바꿔도 동일 목표 유지. Pro 확장 시 “목표 여러 개·팀 공유”로도 이어질 수 있음. |

→ **우선 확장 추천**: (1) 요약/인사이트 **사용자별 일일 제한** 도입, (2) **플레이리스트에 user_id** 연결. 그 다음 Pro 플랜·챕터 등 고급 기능.

## 2. 팀/기업 타깃 (스타트업, 스터디, 마케팅 팀)

### 2-1. 현재 맞게 설정된 부분 ✅

| 기능 | 현재 상태 | 타깃과의 정렬 |
|------|------------|----------------|
| **목표 기반 브리핑** | 개인 목표 → “오늘 볼 영상 N개” | 팀 목표로 확장하면 “팀이 이번 주 볼 영상”에 재사용 가능 |
| **북마크** | 개인별 저장 | “팀 북마크”는 **폴더/팀 개념**만 추가하면 확장 가능한 구조 |
| **Briefing API** | 목표+피드로 랭킹 JSON 반환 | 팀용 브리핑·슬랙 봇에서 같은 API 재사용 가능 |

→ **정리**: 팀 타깃을 위한 **데이터 모델(팀·멤버·공유)**은 아직 없고, **개인용 AI·피드 로직**을 팀 단위로 쓰는 확장이 필요합니다.

### 2-2. 팀 타깃 쪽에서 부족한 부분 · 확장 포인트

| 부족한 점 | 확장 방향 (우선순위) |
|-----------|----------------------|
| **팀/조직 개념 없음** | **팀(workspace) 테이블** 도입: `id`, `name`, `plan`(free/team). **팀 멤버** 테이블: `team_id`, `user_id`, `role`. 가입/초대 플로우(이메일 초대, 링크 가입). |
| **공유 북마크 없음** | `bookmarks`에 `team_id`(nullable) 추가 또는 **팀 전용 북마크 테이블**. “팀 폴더”/“프로젝트별 폴더”로 묶어서 팀원이 같이 볼 수 있게. |
| **팀 브리핑 없음** | 팀 목표(또는 팀 키워드) 저장 → 기존 `rankFeedByGoalsAction`에 **팀 목표** 넘겨서 “이번 주 팀 추천” 생성. 결과를 팀 대시보드·슬랙에 전달. |
| **연동 없음** | **슬랙 봇**: Briefing API + 슬랙 Incoming Webhook으로 “오늘/이번 주 팀 추천 영상” 메시지 전송. **노션**: 북마크·요약을 노션 DB 행으로 넣는 연동(노션 API). |
| **과금/플랜 없음** | 팀 플랜: `team.plan = 'team'`, 인원수·기능 제한. 결제는 Stripe 등으로 팀 단위 구독. |

→ **우선 확장 추천**: (1) **팀·멤버 테이블 + 초대**로 “팀” 존재 만들기, (2) **팀 북마크(공유 폴더)** 또는 기존 북마크에 `team_id` 추가, (3) **팀 목표 → 브리핑 API** 호출해 “팀 추천” 노출. 그 다음 슬랙/노션 연동.

## 3. B2B·크리에이터 타깃 (유튜버, 강사, 미디어사)

### 3-1. 현재 맞게 설정된 부분 ✅

| 기능 | 현재 상태 | 타깃과의 정렬 |
|------|------------|----------------|
| **요약·자막 기반 생성** | `summarizeVideoAction`, `getVideoContext` | 크리에이터 “내 영상 요약/챕터”에 같은 파이프라인 재사용 가능 |
| **프롬프트·품질 평가** | `prompts.ts`, 품질 점수·재생성 | 챕터/요약 품질을 유지한 채 B2B API로 노출 가능 |

→ **정리**: **사용자 대상 앱**은 크리에이터 시나리오를 위해 설계된 건 아니지만, **서버 액션·API**를 그대로 쓰면 “채널 단위 요약/챕터 생성” B2B·API 제품으로 확장할 수 있습니다.

### 3-2. B2B·크리에이터 쪽에서 부족한 부분 · 확장 포인트

| 부족한 점 | 확장 방향 (우선순위) |
|-----------|----------------------|
| **채널/크리에이터 단위 진입점 없음** | **크리에이터 대시보드** 또는 **API 키 발급 화면**: 채널 ID(또는 연결된 계정) 입력 → “내 채널 영상 목록” + “요약/챕터 생성” 버튼 또는 API. |
| **챕터 자동 생성 미구현** | 자막/타임스탬프로 구간 나눈 뒤 Gemini에 “구간별 제목” 생성 (이미 계획된 기능). 이걸 **채널 단위로 배치 실행**하면 B2B 기능이 됨. |
| **공유용 카드/이미지 미구현** | 요약 문장 → Imagen 등 이미지 API로 OG/썸네일 생성. `IMAGE_GENERATION_PROMPTS.md` 활용. **API 엔드포인트**로 제공하면 크리에이터·미디어사가 자동화에 쓸 수 있음. |
| **과금·API 제한 없음** | **API 키** + 사용량(호출 횟수) 저장. 월 구독 또는 호출당 과금. `api_usage` 테이블로 크리에이터/앱별 사용량 집계. |

→ **우선 확장 추천**: (1) **챕터 자동 생성** 기능을 먼저 구현(개인 Pro에도 노출), (2) **/api/summarize**, **/api/chapters** 같은 **공개 API** + API 키 인증으로 B2B 진입, (3) 요약 기반 **이미지 생성 API**는 그다음 단계.

## 4. 통합: 부족한 부분 + 확장 우선순위 (1→2→3 순서)

아래는 개인·팀·B2B를 한 흐름으로 합친 **단일 우선순위** 목록입니다.

| # | 부족한 부분 | 확장 방향 (할 일) | 주 타깃 |
|---|-------------|-------------------|--------|
| **1** | 요약/인사이트/브리핑에 사용자별 제한 없음 | Free: 일 N회 요약·인사이트 제한, 브리핑 주 1회. Pro: 무제한. 로그인 사용자 기준으로 DB/Redis에 일일·주간 사용량 적재 후 제한. | 개인 |
| **2** | 플레이리스트가 사용자별이 아님 | `playlists` 테이블에 `user_id` 컬럼 추가. 저장/조회 시 로그인 사용자만 본인 플레이리스트만 보게. | 개인 |
| **3** | Pro 전용 기능·플랜 개념 없음 | `subscriptions` 또는 `user_plan` 테이블 도입. 챕터/구간 요약, “오늘 한 줄 브리핑” 등 Pro 전용 기능 구현 후 플랜별 분기. | 개인 |
| **4** | 목표(My Focus)가 로컬만 저장 | 로그인 시 목표를 Supabase 등에 저장·동기화. 기기 바꿔도 동일 목표 유지. (이후 팀 목표 확장 시 재사용) | 개인 |
| **5** | 팀/조직 개념 없음 | 팀(workspace) 테이블 + 팀 멤버 테이블. 팀 생성·초대(이메일/링크) 플로우. | 팀 |
| **6** | 공유 북마크 없음 | `bookmarks`에 `team_id`(nullable) 추가 또는 팀 전용 북마크 테이블. 팀 폴더로 묶어 팀원 공유. | 팀 |
| **7** | 팀 브리핑 없음 | 팀 목표 저장 → 기존 `rankFeedByGoalsAction`에 팀 목표 넘겨 “이번 주 팀 추천” 생성. 팀 대시보드·(선택) 슬랙 전달. | 팀 |
| **8** | 슬랙/노션 등 연동 없음 | 슬랙: Briefing API + Incoming Webhook. 노션: 북마크·요약을 노션 DB 행으로 넣는 연동. | 팀 |
| **9** | 팀·개인 과금/플랜 없음 | 팀 플랜(`team.plan`), 인원·기능 제한. Stripe 등으로 팀·개인 구독 결제. | 개인+팀 |
| **10** | 챕터 자동 생성 미구현 | 자막/타임스탬프로 구간 나눈 뒤 Gemini로 구간별 제목 생성. 개인 Pro + B2B 공용. | 개인+B2B |
| **11** | 크리에이터/채널 진입점·API 없음 | 크리에이터 대시보드 또는 API 키 발급. `/api/summarize`, `/api/chapters` 등 공개 API + API 키 인증. | B2B |
| **12** | 공유용 카드/이미지 생성 없음 | 요약 문장 → Imagen 등으로 OG/썸네일 생성. API 엔드포인트로 제공. | B2B |
| **13** | API 사용량·과금 없음 | API 키별 사용량 저장(`api_usage`). 월 구독 또는 호출당 과금. | B2B |

**흐름 요약**: 1~4로 개인(Free/Pro) 기반 다진 뒤 → 5~8로 팀 기능 → 9로 과금 정리 → 10~13으로 B2B·크리에이터 확장.

## 5. 요약 표: 타깃별 정렬도 · 확장 우선순위

| 타깃 | 현재 정렬도 | 가장 중요한 확장 1~2개 |
|------|-------------|-------------------------|
| **개인** | ✅ 높음. 피드·요약·브리핑·북마크·라디오·로그인 이미 있음. | ① 요약/인사이트 **일일 제한**(Free) ② **플레이리스트 user_id** |
| **팀/기업** | ⚠️ 낮음. 팀·공유·연동 없음. | ① **팀·멤버·팀 북마크** ② **팀 브리핑 + (선택) 슬랙** |
| **B2B·크리에이터** | ⚠️ 낮음. 채널/API 전용 화면·과금 없음. | ① **챕터 생성** ② **API 키 + 요약/챕터 API** |

---

## 6. 결론

- **개인 타깃**: 지금 구조가 이미 잘 맞음. **사용자별 제한(Free/Pro)**과 **플레이리스트 개인화**만 보완하면 수익화 시나리오(Free vs Pro)와 정렬됩니다.
- **팀 타깃**: “맞게 설정”된 건 개인용 기능을 **재사용할 수 있는 부분**뿐. **팀·공유·팀 브리핑**을 새로 도입하는 확장이 필요합니다.
- **B2B·크리에이터**: **요약/자막 파이프라인**은 그대로 쓸 수 있음. **챕터·API·과금**을 붙이면 확장 가능합니다.

공모전 서류에는 “현재는 개인 타깃에 최적화돼 있고, 팀·B2B는 위 확장 포인트로 단계적 확장 가능하다”고 쓰면 됩니다.
