# [비용 분석] 포커스 피드 운영 및 고도화 예상 비용 리포트

AI 1인 기업가가 혼자 사용할 경우를 가정하여, 현재 스택과 제안된 고도화 기능을 보수적으로 계산한 월평균 예상 비용입니다.

---

## 2. 유료 전환 시 시나리오 (보수적 추정)

만약 사용량이 많아져 무료 한도를 초과하거나, 더 높은 품질을 위해 유료 모델(Pay-as-you-go)을 사용할 경우의 비용입니다.

### A. AI 요약 및 대화 (Gemini 1.5 Flash 기준)
- **가정**: 매일 50개 항목 분석 + 랭킹 2회 + 대화 20회
- **월간 토큰 사용량**: 약 300만 토큰 (입력), 50만 토큰 (출력)
- **계산**: (3 * $0.075) + (0.5 * $0.3) = 약 $0.375
- **결과**: **약 500원 / 월**

### B. 고품질 음성 브리핑 (Google Cloud TTS Neural 기준)
- **가정**: 매일 3,000자(뉴스 3~5건 요약) 브리핑 생성
- **월간 글자수**: 약 9만자
- **무료 한도**: 100만자까지 무료
- **결과**: **$0 (여전히 무료)**

### C. 데이터 저장 및 통신 (Supabase/Vercel)
- **결과**: 1인 기업가 수준의 텍스트 데이터로는 무료 티어를 벗어나기가 매우 어렵습니다. (수만 개의 북마크가 쌓여도 500MB를 넘기 힘듦)
- **결과**: **$0**

## 3. 핵심 요약: "무료로 가능한가?"

> [!IMPORTANT]
> **결론: 1인 기업가가 개인용으로 사용하기에는 "영원히 $0"에 가깝게 운영 가능합니다.**

- **현재 구조**: Vercel(Next.js) + Supabase + Gemini 무료 티어 조합은 1인 사용 시 비용 발생 지점이 거의 없습니다.
- **가장 큰 변수**: Gemini API의 **분당 요청 제한(Rate Limit)**입니다. 짧은 시간에 수십 개를 한꺼번에 요약하려고 하면 "잠시 후 시도" 메시지가 뜰 수 있습니다. (이 경우 유료 $0.2 내외 결제 시 해결됨)

---

## 4. 비용 최적화 전략 (Scaling Smart)

비용을 최소로 유지하며 성능을 높이는 방법입니다:

1.  **철저한 캐싱**: 한 번 요약된 내용은 Supabase DB에 저장하여 다시 API를 호출하지 않도록 합니다. (현재 이미 구현 중인 방식)
2.  **임베딩 재사용**: Feed Chat 기능을 위해 생성한 벡터 데이터도 DB에 영구 저장하여 중복 연산을 방지합니다.
3.  **모델 이원화**: 단순 데이터 추출은 가장 저렴한 `Gemini 1.5 Flash`를 쓰고, 아주 복잡한 분석이나 전략 제안에만 상위 모델(`Gemini 1.5 Pro`)을 선택적으로 사용하게 설계합니다.

---

**최종 의견**: 1인 기업가로서 이 시스템을 구축했을 때 운영 비용 걱정은 거의 하지 않으셔도 됩니다. 오히려 **"나의 시간 1시간을 아껴주는 가치"**가 월 몇천 원의 잠재적 비용보다 훨씬 크기 때문에, 적극적인 고도화를 추천드립니다.
