# Antigravity QA 로그 안내

브라우저 개발자 도구 콘솔에서 **`[FocusFeed]`** 로 필터하면 Focus Feed 라디오/플레이어 동작을 추적할 수 있습니다.

## 필터 방법

- Chrome/Edge: 콘솔 상단 필터 입력란에 `[FocusFeed]` 입력
- 또는 콘솔에서 "FocusFeed"로 검색

## 로그 포인트 (Radio)

| 로그 메시지 | 의미 |
|------------|------|
| `[Radio] 라디오에 추가` | 피드에서 "라디오에 추가" 클릭. `videoId`, `title`, `hasSummary` |
| `[Radio] 큐에서 제거` | 플레이리스트 서랍에서 휴지통 클릭. `index`, `videoId` |
| `[Radio] 재생 위치 변경` | 다음/이전 버튼 또는 서랍에서 곡 클릭. `index`, `title` |
| `[Radio] AI 요약 저장됨` | 요약 불러오기 성공 후 큐 아이템에 반영됨. `videoId` |
| `[Radio] 플레이어 닫힘 (큐 비움)` | 플레이어 X 버튼. `queueLength` |
| `[Radio] 플레이리스트 서랍 열림` | ListMusic 버튼. `queueLength` |
| `[Radio] 플레이리스트 서랍 닫힘` | 서랍 닫기 |
| `[Radio] AI 요약 가사 뷰 열림` | FileText 버튼. `hasSummary` (이미 요약 있음 여부) |
| `[Radio] AI 요약 가사 뷰 닫힘` | 가사 뷰 닫기 |
| `[Radio] 요약 불러오기 시작` | 가사 뷰에서 "요약 불러오기" 클릭. `videoId` |
| `[Radio] 요약 불러오기 완료` | 요약 API 성공. `videoId` |
| `[Radio] 요약 불러오기 실패` | 요약 API 실패 (console.warn). `videoId`, `message` |
| `[Radio] 미니 영상 켜짐 (인앱 시청)` | Video 버튼으로 우하단 영상 표시 |
| `[Radio] 미니 영상 꺼짐` | Video 버튼으로 영상 숨김 |
| `[Radio] 전체 화면 영상 열림` | Maximize2 버튼으로 전체 화면 모달 열림 |
| `[Radio] 전체 화면 영상 닫힘` | X·백드롭·Escape로 전체 화면 닫힘 |

## 검증 시 확인할 흐름 예시

1. **라디오 추가** → `라디오에 추가` 로그
2. **서랍 열기** → `플레이리스트 서랍 열림`
3. **다른 곡 클릭** → `재생 위치 변경`
4. **가사 뷰 → 요약 불러오기** → `요약 불러오기 시작` → `요약 불러오기 완료` 또는 `실패` → (성공 시) `AI 요약 저장됨`
5. **미니 영상 토글** → `미니 영상 켜짐` / `미니 영상 꺼짐`
6. **플레이어 닫기** → `플레이어 닫힘 (큐 비움)`

이 순서대로 로그가 나오는지 보면 Antigravity QA에서 동작 여부를 빠르게 확인할 수 있습니다.
