# Focus Feed UI/UX 리뷰 & 최신 트렌드 가이드

> UI/UX에 익숙하지 않은 분도 이해하기 쉽게, **현재 상태 → 느낀 점 → 트렌드 설명 → 적용 방법** 순서로 정리했습니다.

---

## 1. 현재 UI/UX 요약

### 1-1. 앱이 하는 일 (한 줄)
유튜브·RSS 피드를 한곳에서 보고, **라디오처럼 오디오 재생 + AI 요약**을 할 수 있는 "Focus Feed" 서비스입니다.

### 1-2. 사용하는 디자인 시스템
- **색상**: Notion 스타일 변수 (`--notion-bg`, `--notion-fg`, `--notion-gray`, `--notion-hover`, `--notion-border`)
- **라이트/다크 모드**: `next-themes`로 지원
- **폰트**: Geist Sans / Geist Mono (Next.js 기본)
- **레이아웃**: 사이드바(데스크톱) + 상단 헤더(모바일) + 하단 고정 플레이어
- **컴포넌트**: Lucide 아이콘, Tailwind CSS, 둥근 모서리(`rounded-lg`, `rounded-2xl` 등)

---

## 2. 디자이너 관점에서의 느낀 점

### 잘 되어 있는 점
1. **일관된 톤**: Notion 팔레트를 쓰면서 라이트/다크가 통일되어 있어서 "한 서비스" 느낌이 난다.
2. **정보 구조**: 전체 피드 → 카테고리/소스 필터 → 개별 피드 아이템 → 라디오 재생 흐름이 명확하다.
3. **접근성**: 버튼에 `aria-label`, `title`이 있어 스크린 리더·키보드 사용자를 고려했다.
4. **기능 완성도**: 플레이리스트 서랍, AI 요약 뷰, 미니/전체 영상 전환이 구현되어 있다.

### 개선하면 좋을 부분 (쉽게 설명)
1. **시각적 계층이 약함**  
   - 제목·부제·버튼·캡션의 "중요도 차이"가 폰트 크기·굵기·색 농도만으로 구분되어 있어, 한눈에 "어디를 먼저 봐야 할지"가 조금 흐릿할 수 있다.
2. **여백과 리듬**  
   - 카드/리스트 간 간격이 비슷해서, "블록 단위"가 명확하지 않다. 여백을 조금 더 단계별로 주면 읽기 편해진다.
3. **플레이어가 기능 위주**  
   - 재생/이전/다음/목록/가사/영상 등 버튼이 많아서, "지금 재생 중인 콘텐츠"가 강조되지 않고 버튼들이 먼저 눈에 들어올 수 있다.
4. **브랜드 정체성**  
   - "F" 로고와 "Focus Feed" 텍스트만으로는 서비스 성격(라디오 + AI 요약)이 바로 와닿지 않는다. 색 한 가지나 작은 그래픽 요소만 더해도 인상이 달라진다.
5. **빈 상태(Empty state)**  
   - 큐가 없을 때 문구는 친절하지만, 일러스트나 아이콘 하나 있으면 "여기에 뭔가 채워진다"는 느낌을 주기 좋다.

## 3. 최신 트렌드 (쉽게 이해하기)

아래는 "2024~2025년에 자주 쓰이는 UI/UX 트렌드"를 **비전공자도 이해할 수 있게** 짧게 정리한 것입니다.

| 트렌드 | 의미 (쉽게) | Focus Feed에 적용 예 |
|--------|-------------|------------------------|
| **글래스모피즘** | 반투명 + 블러로 "유리 뒤" 느낌 | 플레이어 바, 모달 배경을 `backdrop-blur`로 살짝 더 강조 |
| **소프트 UI / 네오모피즘** | 그림자·밝기로 눌린/돌출된 느낌 | 버튼·카드에 `box-shadow`로 살짝 입체감 |
| **대담한 타이포그래피** | 제목을 크고 굵게 해서 한 번에 눈에 들어오게 | "Focus Feed" 메인 타이틀 크기·굵기 강화 |
| **마이크로 인터랙션** | 클릭·호버 시 작은 움직임/색 변화 | 버튼 호버 시 scale 살짝, 재생↔일시정지 아이콘 전환 애니메이션 |
| **카드/리스트 구분** | 카드는 "한 덩어리", 리스트는 "선으로 구분" | 피드 아이템을 카드형으로 감싸거나, 구분선을 더 명확히 |
| **포커스 컬러 1가지** | 서비스마다 "대표색" 하나 정해서 강조 | 재생 버튼·진행 중 표시에 포커스 컬러 사용 (예: 그린/블루) |
| **다크 모드 우선** | 다크를 기본으로 두고 라이트는 대비용 | 이미 다크 지원 중이므로, 다크에서 계층·대비만 살짝 다듬기 |
| **라디오/스트리밍 앱 UX** | 재생 중인 곡(제목·커버)이 가장 크고, 컨트롤은 그 다음 | 하단 플레이어에서 "지금 재생 중" 영역을 넓히고, 버튼은 그 옆/아래 |

---

## 4. 적용 방법 (우선순위별)

### 4-1. 비용 거의 없이 바로 할 수 있는 것 (코드/스타일만)

1. **포커스 컬러 하나 정하기**  
   - 예: 재생 버튼·재생 중 인디케이터에 `emerald-500` 또는 `blue-500` 적용.  
   - `globals.css`에 `--focus-accent: ...` 같은 변수 하나 두고, 재생/활성 상태에만 쓰면 된다.

2. **메인 타이틀 강조**  
   - "Focus Feed"를 `text-4xl` → `text-5xl`, `font-bold` → `font-extrabold` 정도로 키우고,  
   - 서브 텍스트는 `text-(--notion-fg)/60` 유지해 "제목 vs 설명" 계층을 분명히 한다.

3. **플레이어에서 "재생 중" 영역 넓히기**  
   - 현재: 제목 + "n개 · m번째"가 한 줄로 잘림.  
   - 제목을 2줄까지 보이게 하거나, 재생 중인 항목만 배경색/테두리로 살짝 강조해 주면 "지금 이걸 듣고 있다"는 인지가 쉬워진다.

4. **버튼 호버 시 마이크로 인터랙션**  
   - `transition transform hover:scale-105` 같은 걸 재생/일시정지, 리스트 열기 등 주요 버튼에만 적용.  
   - 과하지 않게 0.95~1.05 수준이면 충분하다.

5. **피드 아이템을 카드처럼**  
   - 각 피드 아이템을 `rounded-xl border bg-(--notion-bg)` 같은 박스로 감싸고,  
   - 리스트 사이에 `gap-3` 정도 주면 "한 덩어리씩" 구분이 잘 된다.

### 4-2. 이미지·에셋이 있으면 좋은 것

| 용도 | 설명 | 형식 예 |
|------|------|---------|
| **앱 로고/파비콘** | "F" 대신 브랜드 로고 (라디오+피드 느낌) | SVG 또는 PNG 32x32, 192x192 |
| **빈 상태 일러스트** | "재생 목록이 비어 있을 때" 화면용 단순 일러스트 | SVG (다크/라이트 대응 또는 단색) |
| **재생 중 플레이스홀더** | 큐에 썸네일이 없을 때 쓸 기본 이미지 | 정사각형 비율, 120x120 이상 |
| **오픈 그래프 이미지** | SNS 공유 시 보이는 대표 이미지 | 1200x630 PNG/JPEG |

이미지 없이도 **Lucide 아이콘**(예: `Music2`, `Radio`)만 크게 키워서 빈 상태나 재생 영역에 넣어도 된다.

### 4-3. 나중에 여유 있을 때

- **진행 바(Progress bar)**  
  - 현재 구간 / 전체 길이를 시각적으로 보여주면 "라디오/스트리밍" 느낌이 더 난다.  
  - YouTube API에서 구간 정보를 쓸 수 있으면 적용 가능.

- **재생 중 카드에 웨이브/펄스**  
  - 재생 중인 큐 아이템 옆에 작은 애니메이션(막대기 웨이브 등)을 넣으면 "지금 재생 중"이 더 직관적이다.

- **키보드 단축키 안내**  
  - 스페이스: 재생/일시정지, J/K: 이전/다음 등 간단한 단축키를 도움말이나 툴팁으로 노출하면 파워유저 경험이 좋아진다.

## 5. 디자인 토큰 정리 (참고용)

지금 쓰는 것에 **포커스 컬러**만 하나 더 두면, 나중에 테마 확장할 때 편하다.

```css
/* globals.css 예시 추가분 */
:root {
  /* 기존 --notion-* 유지 */
  --focus-accent: #10b981;        /* emerald-500, 재생/활성 강조 */
  --focus-accent-muted: rgba(16, 185, 129, 0.15);
}
.dark {
  --focus-accent: #34d399;        /* emerald-400 */
  --focus-accent-muted: rgba(52, 211, 153, 0.12);
}
```

- **재생 버튼 배경**: `bg-(--focus-accent)`  
- **재생 중인 큐 행**: `bg-(--focus-accent-muted)`  
- **"연결됨" 뱃지**: 기존 emerald 유지 또는 `--focus-accent`로 통일  

이렇게 하면 "초록 = 재생/활성"이라는 일관된 언어가 생긴다.

## 6. 한 줄 요약

- **현재**: Notion 스타일로 정돈되어 있고, 기능과 접근성은 잘 갖춰져 있음.  
- **보완 포인트**: 시각적 계층(제목 강조, 포커스 컬러), 플레이어에서 "재생 중" 강조, 여백/카드감, 작은 인터랙션.  
- **트렌드 반영**: 포커스 컬러 1가지, 대담한 타이틀, 라디오 앱처럼 "재생 중"이 가장 크게 보이게, 마이크로 인터랙션.  
- **에셋**: 로고/파비콘, 빈 상태용 아이콘 또는 일러스트, (선택) OG 이미지·재생 플레이스홀더.

원하시면 위 항목 중 "우선 적용할 1~2가지"를 정해 주시면, 그에 맞춰 **어디를 어떻게 수정하면 되는지(파일/클래스명 수준)** 로 구체적인 수정안을 적어 드리겠습니다.
