# AI Collaboration Playbook

## 역할 분담

### Antigravity

- 큰 그림 설계와 우선순위 결정을 담당합니다.
- 작업을 `즉시 수정`, `이번 사이클`, `후속 개선`으로 나눕니다.
- 브라우저 기반 검증 시나리오와 완료 기준을 정의합니다.
- 최종 QA 스크린샷과 확인 결과를 남깁니다.

### Cursor

- 코드베이스 점검과 실제 구현 수정을 담당합니다.
- 보안, 안정성, 유지보수성 문제를 우선 해결합니다.
- lint, build, 동작 검증으로 회귀 여부를 확인합니다.
- 변경 사항과 남은 위험 요소를 문서화합니다.

## 작업 순서

1. Antigravity가 이번 사이클의 목표와 완료 기준을 정리합니다.
2. Cursor가 영향 범위를 확인하고 안전한 수정 순서를 제안합니다.
3. Cursor가 작은 단위로 수정하고 중간 검증 결과를 공유합니다.
4. Antigravity가 브라우저 테스트 체크리스트와 증빙을 업데이트합니다.
5. 둘 다 남은 리스크와 다음 액션을 합의한 뒤 다음 사이클로 넘어갑니다.

## 사이클 기본 순서

1. 보안
2. 안정성
3. 사용자 경험
4. 문서화

## 브라우저 검증 체크리스트

- 메인 피드가 정상적으로 렌더링되는지 확인
- **소식통**: 유튜브 최신 / RSS·뉴스 최신 블록이 각각 표시되고, 빈 블록 시 안내 문구가 나오는지 확인
- **카테고리 필터**: 전체·AI·자기계발·개발·뉴스 전환 시 항목이 해당 카테고리만 보이는지, URL `?category=` 반영·사이드바 하이라이트가 맞는지 확인
- 필터 추가/삭제가 정상 동작하는지 확인
- 새로고침 버튼 동작 후 피드가 다시 로드되는지 확인
- 빈 결과 상태 메시지가 자연스러운지 확인
- 외부 링크가 새 탭에서 열리는지 확인
- 썸네일이 실패하더라도 레이아웃이 깨지지 않는지 확인
- **라디오 플레이어**: "라디오에 추가" → 하단 플레이어 표시·재생/일시정지/다음/이전/닫기, 탭 전환 후 오디오 유지
