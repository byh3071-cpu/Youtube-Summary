# 나만의 AI 유튜브 라디오 (YouTube AI Aggregator) Cursor Handoff

이 문서는 Antigravity와 Cursor가 협력하여 애플리케이션을 개발하기 위한 지침서입니다. Cursor 채팅창에 **[현재 진행 단계]** 의 블록을 복사해서 붙여넣고 개발을 진행하세요.

---

## [Phase 5] 고급 라디오 UI & UX 고도화 (현재 진행 단계 / 복사 영역)

**배경 (Context)**
Phase 1~4를 거쳐 유튜브/RSS 피드 연동, AI 3줄 요약, 리팩토링 및 백그라운드 오디오 라디오 큐 연동까지 기초 토대가 모두 완성되었습니다. 이제는 사용성이 밋밋한 하단 라디오 플레이어에 **스포티파이(Spotify)급의 음악 스트리밍 앱 UX**를 적용하여 완성도를 끌어올립니다.

**목표**
사용자가 현재 재생 대기열(플레이리스트)을 직관적으로 확인/제어하고, 오디오를 들으면서 요약된 AI 텍스트를 마치 '가사'처럼 보며, 필요한 경우 외부 유튜브 이탈 없이 앱 내부에서 영상을 볼 수 있도록 플레이어를 확장합니다.

**우선순위 (개발 대상)**
1. **재생 대기열(Playlist Queue) 서랍 UI**:
   - 하단 `FloatingRadioPlayer.tsx` 우측에 "플레이리스트 보기" 버튼(예: ListMusic 아이콘)을 추가하세요.
   - 버튼 클릭 시 하단에서 슬라이드업(Slide-up) 또는 우측에서 튀어나오는 서랍(Drawer) 모달 형태로 `state.queue`의 전체 목록을 보여주세요.
   - 목록 내용: 재생 중인 항목 시각적 강조, 개별 항목 삭제 버튼, 리스트 클릭 시 해당 아이템으로 즉시 점프(`setCurrentIndex`).

2. **AI 요약 가사 뷰 (Lyrics View)**:
   - 플레이어 내부에 "AI 요약 보기" 버튼을 추가하고, 클릭 시 재생 중인 현재 영상의 **AI 3줄 요약** 텍스트를 팝업이나 펼쳐지는 영역에 노출하세요.
   - 이를 위해 기존 큐 아이템 상태에 요약 데이터를 저장할 수 있도록 Context State(`RadioQueueItem` 인터페이스)에 `summary?: string[]` 속성을 추가하고, 피드에서 큐로 넘길 때 데이터를 전달해 주세요.

3. **인앱(In-app) 영상 플레이어 미니 모드**:
   - 현재 오디오 전용 플레이를 위해 1x1 픽셀 투명으로 숨겨둔 iframe 영역(`id="yt-radio-player-host"`)의 화면 크기를 조작합니다.
   - 사용자가 "영상도 같이 보기" 토글 버튼을 누르면, 이 iframe 요소가 비디오를 볼 수 있는 미니 팝업창(PIP 느낌)이나 플레이어 상단의 시각적 영역으로 애니메이션오 함께 확장되어 **웹 앱 내부에서 영상을 시청**할 수 있게 하세요. Tailwind CSS 클래스 토글링으로 `w-px h-px opacity-0`에서 적절한 크기(`w-80 h-48` 등)로 변경하면 됩니다.

**핵심 주의사항**
- 추가되는 확장 UI(플레이리스트 서랍, 가사창, 미니 비디오 팝업)가 디자인을 해치지 않도록 기존의 다크 톤 TailwindCSS(`--notion-bg`, `--notion-fg`, 둥근 모서리 등) 스타일 가이드를 일관되게 적용하세요.
- 전역 상태 관리 Context(`RadioQueueContext.tsx`) 수정 시 어긋남이 발생하지 않도록 주의하고, 불필요한 전체 리렌더링이 없도록 최적화(`useMemo`, `useCallback`)를 신경써주세요.

**완료 기준**
- [x] 하단 플레이어에서 서랍 형태로 플레이리스트 목록 조회 및 재생 제어가 작동함.
- [x] 재생 중인 아이템의 AI 요약 내용을 스포티파이 가사 뷰처럼 깔끔하게 확인할 수 있음.
- [x] 유튜브 링크 클릭 시 외부 새 창 이탈 없이, 앱 내부 플레이어 모달에서 실제 비디오 시청 토글이 가능함.
